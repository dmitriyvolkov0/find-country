# 🎮 Промпт для Qwen CLI: Разработка VK Mini App гео-викторины "Map It!" на React JS

## 📋 Контекст задачи
Ты — senior React-разработчик, специализирующийся на VK Mini Apps. Твоя задача: создать кроссплатформенную игру-викторину "Угадай страну на карте" с интеграцией в экосистему VK, используя CDN-импорты и без процесса сборки.

## 🎯 Цель игры
Пользователю показывается название страны → он кликает по этой стране на 3D-глобусе → система проверяет ответ, начисляет очки и сохраняет прогресс в VK Storage.

---

## ⚙️ Технические ограничения (СТРОГО СОБЛЮДАТЬ)

### Стек технологий
- React 18 через CDN (esm.sh)
- react-globe.gl для 3D-визуализации глобуса
- Tailwind css
- Zustand для управления состоянием игры (подключать через CDN)
- VK Bridge для взаимодействия с платформой VK
- VK Storage для сохранения прогресса (вместо localStorage)
- Babel Standalone для JSX-компиляции в браузере
- D3-модули для цветовых шкал (опционально)

### Интеграция с VK
- Инициализировать VK Bridge при старте приложения
- Использовать VK Storage для сохранения:
  - общего счёта и рекордов
  - прогресса по уровням/регионам
  - статистики правильных ответов
- localStorage разрешён ТОЛЬКО для локальных настроек:
  - включение/выключение звука
  - выбор языка интерфейса
  - графические настройки (качество анимаций)
- Обрабатывать события VK Bridge: viewportChanged, locationChanged

## 📱 Адаптивность и кроссплатформенность

### Требования к отображению
- Интерфейс должен корректно масштабироваться на:
  - мобильных устройствах (320px–480px ширина)
  - планшетах (481px–1024px)
  - десктопах (1025px+)
- Глобус должен занимать доступную область с учётом safe-area-inset на iOS
- UI-элементы не должны перекрывать интерактивную зону глобуса на малых экранах

### Управление
- Desktop: клик мышью, ховер-эффекты, клавиатурная навигация
- Mobile: тап-жесты, увеличенные хит-области, отсутствие ховер-зависимостей
- Поддержка мультитач для масштабирования/вращения глобуса

### Производительность
- Оптимизировать рендеринг для мобильных GPU
- Ленивая загрузка ресурсов (текстуры, GeoJSON)
- Динамическое снижение детализации на слабых устройствах (определение через navigator.hardwareConcurrency или VK Bridge device info)

## 🎮 Геймплей: функциональные требования

### Состояния игры
Реализовать конечный автомат: старт → вопрос → обратная связь → завершение → повтор.

### Логика раунда
1. Игрок запускает игру через VK Mini App интерфейс
2. Система выбирает страну случайным образом из отфильтрованного списка
3. Отображается задание с визуальным таймером
4. Обработка клика/тапа по глобусу:
   - Попадание: визуальный и тактильный отклик (если поддерживается), начисление очков
   - Промах: плавный полёт камеры к правильной стране, показ подсказки
5. Автоматический переход к следующему вопросу
6. После завершения раунда — сохранение результатов в VK Storage и показ статистики

### Визуальная обратная связь
- Динамическое изменение стиля полигонов в зависимости от состояния
- Анимация выделения выбранной страны
- Плавные переходы между состояниями интерфейса

### Дополнительные механики (по приоритету)
- Высокий: прогресс-бар таймера, индикатор текущего вопроса, вибрация при ошибке (через VK Bridge)
- Средний: система достижений, сохранение в VK Storage, адаптивная сложность
- Низкий: выбор региона/темы, звуковые эффекты, социальная интеграция (поделиться результатом)

## 🧱 Компонентная структура

Организовать код логически внутри одного файла:
- Корневой компонент приложения с инициализацией VK Bridge и Zustand store
- Компонент глобуса с пропсами для визуализации и обработчиками событий
- Слой UI с адаптивной вёрсткой поверх canvas
- Презентационные компоненты для каждого состояния игры
- Кастомные хуки:
  - useVKStorage — обёртка над VK Storage с обработкой ошибок
  - useGameEngine — управление состоянием, таймерами, логикой вопросов
  - useResponsive — хук для определения характеристик устройства
- Блок доступности: ARIA-атрибуты, поддержка клавиатуры, достаточная контрастность

- Производительность: стабильные 30+ FPS на средних мобильных устройствах

## 💡 Рекомендации по реализации

- Инициализировать VK Bridge в useEffect с обработкой ошибок
- Создать абстракцию useVKStorage с методами get/set и fallback-логикой
- Использовать Zustand store для централизованного управления состоянием игры
- Реализовать адаптивную логику через matchMedia или хук useResponsive
- При сохранении в VK Storage учитывать лимиты хранилища и обрабатывать квоты
- Для мобильных устройств упростить визуальные эффекты при низкой производительности
